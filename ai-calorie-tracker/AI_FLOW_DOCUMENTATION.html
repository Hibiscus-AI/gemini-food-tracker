Hibiscus Health — AI Food Tracker

AI-powered food identification and nutrition calculation flow for the mobile app

API v3.0.0 | Backend Team Reference

How It Works (Overview)

The AI handles two things only: identifying what's in the photo, and looking up accurate nutrition data. Everything else (saving meals, history, UI) is handled by the backend/frontend teams.

Step 1 — Gemini AI

User Snaps a Photo

Gemini identifies the food, lists each ingredient, and estimates grams per ingredient. Also suggests hidden ingredients (spices, oils, sauces) that aren't visible.

↓
Step 2 — Nutrition Waterfall

Look Up Nutrition Per Ingredient

For each ingredient, check databases in order: IFCT (Indian) → USDA (global) → Local DB → Gemini guess. First match wins.

↓
Step 3 — Calculate

Total Calories & Macros

Each ingredient: grams × (cal per 100g / 100). Sum all = total meal nutrition. Return ingredient chips, breakdown table, and totals.

↓
Step 4 — User Edits

User Adjusts Ingredients or Quantity

User can add/remove/edit ingredients or change grams. App calls /recalculate — instant update, no re-scan needed.

Why IFCT first? Indian foods measured in Indian context — ghee, paneer, dal values reflect actual Indian cooking. USDA measures US produce/cooking styles. For Indian cuisine, IFCT is more accurate.

API Endpoints (for Backend/Frontend Teams)

1. POST /analyze — Scan Food Photo

POST /analyze Content-Type: multipart/form-data file: image file (required, max 10MB) serving_size: "bowl" | "plate" | "cup" | "piece" (optional) num_servings: 1.5 (optional) portion_grams: 350 (optional, overrides serving_size)

Response includes:

FieldTypeDescription
successboolDid the request complete
is_foodboolfalse if selfie/screenshot/non-food image
food_namestring"Chicken Biryani", "Egg Tofu Salad", etc.
confidencefloat0.0 to 1.0 — how sure Gemini is
confidence_warningstring?Warning if confidence < 60%
ingredient_chipsarray[{"name": "Rice", "cal": 260, "grams": 200}] — ready for UI chips
suggested_ingredientsarray?["turmeric", "cumin", "salt"] — hidden ingredients user can add
ingredient_breakdownarrayFull nutrition per ingredient (cal, protein, carbs, fat, fiber, source)
nutrition_per_servingobjectTotal: {calories, protein_g, carbs_g, fat_g, fiber_g}
nutrition_per_100gobjectPer 100g reference values
estimated_portion_gfloatTotal estimated weight in grams
data_sourcestring"verified" | "verified+fallback" | "local_db" | "ai" | "label"
tipsstring?Brief health tip from Gemini
warningsarray?Any issues (low confidence, clamped grams, etc.)
dishesarray?Only present when multiple dishes detected on plate

2. POST /recalculate — Update After User Edits

Call this when user adds/removes/changes ingredient grams. No image needed. Instant response.

POST /recalculate Content-Type: application/json { "ingredients": [ { "name": "basmati rice", "grams": 200 }, { "name": "chicken", "grams": 150, "cooking_method": "fried" } ], "portion_grams": 350 // optional override }

Returns the same nutrition fields: ingredient_chips, ingredient_breakdown, nutrition_per_serving, nutrition_per_100g, data_source.

3. GET /health — Server Status

GET /health // Returns: model, DB counts, serving sizes, status

4. GET /serving-sizes — Available Presets

GET /serving-sizes // Returns: {"bowl": 250, "plate": 350, "cup": 240, "piece": null, ...}

Edge Cases Handled

Things the frontend team should be aware of:

📷
Non-food image (selfie, screenshot, random object) Returns is_food: false with error message. Show the error to user, don't show nutrition.
🏚
Packaged food with nutrition label AI reads the label directly. Returns data_source: "label". Most accurate — no estimation involved.
⚠️
Blurry / dark photo Returns low confidence (<60%). Check confidence_warning field and show it to user.
🍲
Multiple dishes on plate Returns dishes array with separate breakdown per dish. Flat fields show combined totals.
☕
Beverages (coffee, juice, lassi) Auto-detected. dish_type: "beverage". Grams represent ml (density ~1 for most drinks).
🍳
Cooking method matters "Fried egg" gets a fat multiplier vs "boiled egg". Cooking method is preserved, not stripped.
⌛
AI timeout If Gemini takes >15s, returns error: "AI analysis timed out". User should retry with clearer photo.
💾
Image too large (>10MB) Returns HTTP 413. Frontend should compress or resize before uploading.
💡
Ingredient not in any database Falls through to Gemini AI estimate. source: "ai" on that ingredient. Less accurate but never returns empty.
